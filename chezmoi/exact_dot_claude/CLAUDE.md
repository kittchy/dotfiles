# CLAUDE.md

## Conversation Guidelines

- 常に日本語で会話する
- 技術的な説明は具体例を含める
- 不明点があれば実装前に質問する

## Development Philosophy

### Be Able to Explain

- 最初からコードを書くのではなく、まずは仕様を理解し、説明できるようにする
- 最初から全てを実装するのではなく、必要最低限の機能を作ることに専念する
- 必要のない機能を勝手に追加しない
- トレードオフを明示的に説明する（パフォーマンス vs 可読性、シンプルさ vs 拡張性など）

### Incremental Development

- 小さく動くものを先に作り、段階的に改善する
- 各ステップで動作確認を行う
- リファクタリングは動作確認後に行う

**必須**: フォルダのファイルはUTF-8なので、UTF-8で出力してください。(SHIFT-JISでは文字化けが発生してしまいます)

### Code Review

- コードを実装した際には、必ずcode-reviewスキルをバックグラウンドで実行する
- レビュー結果を確認し、指摘事項があれば対応する
- レビューは実装完了後すぐに実行すること

### Code Style

- コード内のコメントやdocstringは英語で書く
- 日本語はコード内に含めない（文字化け防止）
- README.md や 説明は日本語で書くこと

### タスクの処理の仕方

- しっかりweb searchを行い、正確な情報を提供する
- 個人的な思い込みは提供せず、リファレンスを必ず確認する
