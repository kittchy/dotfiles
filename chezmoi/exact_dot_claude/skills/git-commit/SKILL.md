# Gitで変更をcommit

- git statusを使って変更箇所を理解し、コミットを作成する
- commit の prefixは以下の通り
  - "[FIX]" : バグや不具合の修正
  - "[FEAT]" : 新機能の追加
  - "[DOC]" : ドキュメントの更新や改善
  - "[STYLE]" : コードフォーマットの修正、UIのみの変更
  - "[REFACT]" : パフォーマンスの改善なしのコードの改善
  - "[TEST]" : テストの追加や改善
  - "[PREF]" : パフォーマンスの改善
  - "[CHORE]" : ビルドプロセスの変更や改善
  - "[WIP]" : 作業中
  - "[DEL]" : コードの削除
- commit時は `git commit -m "[FIX] <message>"` のように `-m` オプションを使うこと
